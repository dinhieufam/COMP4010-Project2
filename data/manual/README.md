# Manual Institution and Country Feedback

This folder stores human feedback for cleaning institution names and institution-to-country mappings.

The goal is to make affiliation cleanup **reproducible**: every human correction is saved in CSV and can be reapplied whenever the data pipeline is rebuilt.

## Files

### 1. `institution_aliases.csv`

Use this file to map a raw/noisy institution string to a clean canonical institution name.

Schema:

```csv
raw_institution,canonical_institution,action,status,reviewer,notes
```

Columns:

- `raw_institution`: Exact institution string as it appears in the processed/interim data.
- `canonical_institution`: Clean institution name to use in the dashboard. Leave blank for `drop` or `unsure` if no canonical name is known.
- `action`: One of `keep`, `merge`, `drop`, `split`, `unsure`.
- `status`: One of `pending`, `approved`, `rejected`, `needs_discussion`.
- `reviewer`: Name/initials of the human reviewer.
- `notes`: Optional explanation.

Allowed actions:

| action | Meaning |
|---|---|
| `keep` | Raw name is already correct/canonical. |
| `merge` | Raw name should be replaced by `canonical_institution`. |
| `drop` | Raw string is not an institution and should be removed. |
| `split` | Raw string contains multiple institutions; list outputs in `institution_splits.csv`. |
| `unsure` | Reviewer cannot decide yet; leave unchanged for now. |

Example:

```csv
raw_institution,canonical_institution,action,status,reviewer,notes
University of Toronto.,University of Toronto,merge,approved,HD,Trailing punctuation
University of Toronto1,University of Toronto,merge,approved,HD,PDF superscript artifact
Co-corresponding Authors,,drop,approved,HD,Not an institution
University of Toronto1 Vector Institute2 NVIDIA3,,split,approved,HD,Compound PDF affiliation
```

---

### 2. `institution_splits.csv`

Use this file when one raw institution string should become multiple canonical institutions.

Schema:

```csv
raw_institution,canonical_institution
```

Rules:

- Add one row per canonical output institution.
- The `raw_institution` value should exactly match the raw string in `institution_aliases.csv` with `action=split`.
- Do not add country columns here; country corrections belong in `institution_country_overrides.csv`.

Example:

```csv
raw_institution,canonical_institution
University of Toronto1 Vector Institute2 NVIDIA3,University of Toronto
University of Toronto1 Vector Institute2 NVIDIA3,Vector Institute
University of Toronto1 Vector Institute2 NVIDIA3,NVIDIA
```

---

### 3. `institution_country_overrides.csv`

Use this file to correct or approve country mappings for institutions.

Schema:

```csv
match_type,institution,country_iso2,status,reviewer,notes
```

Columns:

- `match_type`: Either `canonical` or `raw`.
- `institution`: Institution string to match.
  - If `match_type=canonical`, this should be a clean canonical institution name.
  - If `match_type=raw`, this should be an exact raw institution string.
- `country_iso2`: ISO-2 country code such as `US`, `CN`, `GB`, `CA`, `CH`, `HK`.
- `status`: One of `pending`, `approved`, `rejected`, `needs_discussion`.
- `reviewer`: Name/initials of the human reviewer.
- `notes`: Optional explanation.

Precedence when applying country feedback:

1. Approved `raw` override
2. Approved `canonical` override
3. OpenAlex/OpenReview country
4. PDF heuristic country
5. `Unknown`

Use `raw` overrides for branch-specific multinational cases.

Example:

```csv
match_type,institution,country_iso2,status,reviewer,notes
canonical,University of Toronto,CA,approved,HD,Canadian university
canonical,Stanford University,US,approved,HD,US university
canonical,Tsinghua University,CN,approved,HD,Chinese university
canonical,Meta,US,approved,HD,Default company mapping
raw,Meta (Israel),IL,approved,HD,Raw string explicitly says Israel
raw,Meta Reality Labs Zurich,CH,approved,HD,Zurich branch
raw,Google DeepMind (United Kingdom),GB,approved,HD,UK branch
```

### 4. `institution_review_queue.csv`

This is a work queue for human review. It can be generated by scripts and edited by reviewers.

Schema:

```csv
raw_institution,suggested_canonical,suggested_country_iso2,paper_count,source_mix,problem_flags,status,reviewer,decision_notes
```

Columns:

- `raw_institution`: Raw institution string needing review.
- `suggested_canonical`: Auto-suggested canonical institution, if any.
- `suggested_country_iso2`: Auto-suggested country code, if any.
- `paper_count`: Number of unique papers affected.
- `source_mix`: Sources that produced this institution, e.g. `pdf_text`, `openalex_hash`, `openreview`, or a pipe-separated mix.
- `problem_flags`: Pipe-separated flags such as `alias_candidate`, `artifact_candidate`, `country_conflict`, `source_disagreement`, `high_frequency`.
- `status`: One of `pending`, `approved`, `rejected`, `needs_discussion`, `migrated`.
- `reviewer`: Name/initials of reviewer.
- `decision_notes`: Reviewer notes or where the decision was moved.

Important:

- This file is a queue, not the final source of truth.
- Once a decision is made, copy the durable correction into one or more of:
  - `institution_aliases.csv`
  - `institution_splits.csv`
  - `institution_country_overrides.csv`
- Then mark the queue row as `migrated` or `approved`.

Example:

```csv
raw_institution,suggested_canonical,suggested_country_iso2,paper_count,source_mix,problem_flags,status,reviewer,decision_notes
University of Toronto1,University of Toronto,CA,12,pdf_text,alias_candidate,pending,,
Meta Reality Labs Zurich,Meta Reality Labs,CH,4,pdf_text,country_conflict,pending,,
Co-corresponding Authors,,Unknown,2,pdf_text,artifact_candidate,pending,,
```

## Recommended Review Workflow

1. Start with `institution_review_queue.csv`.
2. Prioritize rows with high `paper_count` and flags like `high_frequency`, `country_conflict`, and `artifact_candidate`.
3. For alias/duplicate names, add decisions to `institution_aliases.csv`.
4. For compound strings containing multiple organizations, add `action=split` in `institution_aliases.csv` and add split outputs to `institution_splits.csv`.
5. For country corrections, add rows to `institution_country_overrides.csv`.
6. Mark queue rows as `migrated` once durable corrections have been copied to the source-of-truth CSVs.
7. Rebuild the pipeline/app data after applying feedback.

## Status Values

Use these consistently:

| status | Meaning |
|---|---|
| `pending` | Needs human review. |
| `approved` | Reviewer approved this row as a source-of-truth correction. |
| `rejected` | Suggested correction is wrong. |
| `needs_discussion` | Ambiguous; needs another reviewer or evidence. |
| `migrated` | Queue item has been copied into a durable feedback file. |

## Country Codes

Use ISO-2 country codes, for example:

```text
US, CN, GB, CA, HK, SG, CH, DE, FR, IL, KR, JP, AU, IN, NL, IT
```

If country is unknown, leave `country_iso2` blank rather than writing a guess.

## Best Practices

- Prefer exact raw strings copied from the data to avoid accidental mismatches.
- Do not merge branch-specific names if the branch country matters.
  - Good raw override: `Meta Reality Labs Zurich → CH`
  - Good canonical default: `Meta → US`
- Use `drop` aggressively for obvious PDF artifacts, emails, and non-organization fragments.
- Keep notes for questionable decisions so future reviewers understand the rationale.
- Do not edit generated processed parquet files directly; edit these CSVs and rerun the feedback pipeline.
